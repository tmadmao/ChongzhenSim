# 版本更新日志

## v0.1.5 (2026-03-24)

### 新增功能
- 实现了完整的游戏设置功能，包括存档管理、模式切换、LLM 配置和主题选择
- 在底部栏添加了设置按钮，使用 Lucide 的 Settings 图标
- LLM 客户端现在优先从用户设置中读取 API 配置，提高了灵活性

### 界面优化
- 修复了设置面板弹出时其他界面全黑的问题，现在使用半透明背景效果
- 移除了底部栏的存档按钮，避免功能重复
- 优化了设置面板的样式，使用了统一的模态框样式

### 技术改进
- 修复了 TypeScript 编译错误，确保代码类型安全
- 改进了 LLM 客户端的配置管理逻辑
- 优化了组件的懒加载实现

## v0.1.4 (2026-03-23)

### 性能优化
- 配置了 Vite 构建的 manualChunks，将 react、react-dom 等依赖划分为单独的 chunk
- 调整了 chunkSizeWarningLimit 为 1000KB，减少构建时的超限警告
- 实现了代码分割，使用 React.lazy 和 Suspense 对主要界面组件进行懒加载

### 代码质量
- 修复了 51 个 TypeScript 编译错误，确保代码类型安全
- 优化了 LLM 客户端的类型定义和错误处理

### 依赖管理
- 确保了所有依赖包的版本兼容性
- 添加了必要的类型定义文件

## v0.1.3 (2026-03-22)

### 功能改进
- 实现了全局日志系统，支持 info, warn, error, debug 级别
- 在 GameEngine 和 llmClient 的关键异步方法前后添加了日志记录
- 支持在生产环境下关闭 Debug 级别的日志

### 代码重构
- 将项目中散落在各个 System 里的硬编码数值系数抽离到 src/config/gameConfig.ts
- 修改了原有的 System 文件，使其从配置文件中导入参数
- 确保了游戏逻辑的一致性

## v0.1.2 (2026-03-21)

### 项目初始化
- 初始化了 GitHub 仓库
- 设置了项目的基本结构和依赖
- 配置了 TypeScript 和 Vite 构建系统

## v0.1.1 (2026-03-20)

### 初始版本
- 实现了游戏的基本框架
- 添加了核心游戏系统和组件
